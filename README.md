## Этап 4. Сервис рекомендаций

### Описание сервиса

`recommendations_service.py` — FastAPI-микросервис, принимающий запросы с `user_id` и возвращающий список рекомендованных треков с учётом:

- **офлайн-рекомендаций** (персональных или дефолтных),
- **онлайн-истории** пользователя.



### Стратегия смешивания онлайн- и офлайн-рекомендаций

Сервис реализует смешивние рекомендаций в три шага:

**1. Онлайн-рекомендации (есть история взаимодействий пользователя с сервисом)**  
Берётся история последних действий пользователя из `EventStore` (хранится в памяти).
Для каждого недавно прослушанного трека ,начиная с самого последнего, запрашиваются
похожие треки из  `similar.parquet`.Убираются дубли. И сохраняется порядок релевантности

**2. Офлайн-рекомендации (персонализированные или дефолтные)**  
 Если пользователь есть в `recommendations.parquet`, возвращаются его персональные
  рекомендации, ранжированные CatBoost-моделью. Иначе возвращается глобальный топ-100 популярных треков из `top_popular.parquet`.

**3. Финальное смешивание рекомендаций**  
Онлайн-рекомендации занимают первые позиции (очевидно, что свежий онлайн контекст гораздо важнее). Затем офлайн рекомендации заполняют остаток до K, пропуская уже добавленные треки.

```
результат = [онлайн-рекомендации] + [офлайн-рекомендации, которых нет в онлайн]
```

### Запуск сервиса

**Требования:**  установленные зависимости.

```bash

pip install fastapi uvicorn pandas pyarrow

# запуск из корня репозитория
uvicorn recommendations_service:app --host 0.0.0.0 --port 8000
```

> **P.s.** при первом запуске сервис загружает `recommendations.parquet` (весит больше гигабайта и это длится 1-2 минуты.

> нужно дождаться `Application startup complete` в терминале или проверить `/health`.

Пример запроса (curl):
```bash
curl -X POST "http://127.0.0.1:8000/recommendations?user_id=4&k=10"
```

## Инструкции для тестирования сервиса

### Шаги

1. в отдельном терминале запускаем сервис:
   ```bash
   uvicorn recommendations_service:app --host 0.0.0.0 --port 8000
   ```

2. Дожидаемся готовности (описано выше как).

3. в другом терминале запуск тестов:
   ```bash
   python test_service.py
   ```

4. вывод тестов сохранён в `test_service.log`.



### Сохранение вывода в лог-файл

```bash
python test_service.py 2>&1 | tee test_service.log
```
